# @eproject/components

A React component library for charts, built on [Apache ECharts](https://echarts.apache.org/).

Components are built once with simple, readable settings — then anyone on the team can
customize them, add new ones, or wire in new chart types.

## Setup

### 1. Install Node.js

Download and install from https://nodejs.org/ (version 18 or later).
npm comes bundled with it — no separate install needed.

To verify, open a terminal and run:

```bash
node --version
npm --version
```

Both should print a version number.

### 2. Open the project in Cursor

### 3. Open a terminal in Cursor (Ctrl + `) and run:

```bash
npm install
```

This downloads all required packages. You only need to do this once (or again when
someone adds new packages).

### 4. Start the visual playground

```bash
npm run dev
```

Opens **http://localhost:6006** in your browser — this is Storybook (see next section).

---

## Storybook — the visual playground

Storybook is a local tool (runs on your machine, not in the cloud) that shows every
component with interactive controls.

### What you see

- **Left sidebar** — list of all components (e.g. Charts > LineChart > Basic)
- **Main area** — the selected component rendered live
- **Controls panel** (bottom) — text fields, dropdowns, checkboxes, color pickers
  for every setting — change anything and the chart updates in real-time

### Important: Storybook is read-only

Storybook is a **preview tool**, not an editor. Any changes you make in the Controls panel
are **temporary** — they reset when you refresh the page. They do **not** modify the code.

To actually save changes, use Cursor:

1. Tweak values in Storybook until you like the result
2. Open the corresponding file in Cursor (e.g. `stories/PieChart.stories.tsx`)
3. Edit the values there (or ask Cursor AI to do it for you)

**Storybook to experiment, Cursor to save.**

### Sharing without Node.js

Build a static version anyone can open in a browser:

```bash
npm run build-storybook
```

Creates a `storybook-static/` folder. Open `index.html` from there — no install required.

## Components

### LineChart

Line charts, area charts, and multi-series comparisons.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | — | Chart title at the top center |
| `xLabels` | `string[]` | **required** | Category labels for the X axis |
| `series` | `LineSeriesConfig[]` | **required** | One or more data series (see below) |
| `showTooltip` | `boolean` | `true` | Show info popup on hover |
| `showLegend` | `boolean` | `true` | Show the legend |
| `variant` | `"default"` / `"compact"` / `"dashboard"` | `"default"` | Layout preset (see [Layout variants](#layout-variants)) |
| `colors` | `string[]` | built-in palette | Override the color palette |
| `height` | `number` or `string` | `400` | Chart height in px (or CSS value like `"100%"`) |
| `width` | `number` or `string` | `"100%"` | Chart width |
| `loading` | `boolean` | `false` | Show a loading spinner |
| `className` | `string` | — | CSS class for the container |

#### Series configuration (LineSeriesConfig)

Each item in the `series` array describes one line:

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `string` | **required** | Label (shown in legend and tooltip) |
| `data` | `number[]` | **required** | Values — must match the length of `xLabels` |
| `color` | `string` | auto | Line color, e.g. `"#ee6666"` or `"red"` |
| `smooth` | `boolean` | `false` | Curved line instead of straight |
| `lineWidth` | `number` | `2` | Line thickness in px |
| `showArea` | `boolean` | `false` | Fill area under the line |
| `areaOpacity` | `number` | `0.3` | Area fill opacity (0 = invisible, 1 = solid) |

#### Examples

**Simple line chart:**

```tsx
<LineChart
  title="Monthly Revenue"
  xLabels={["Jan", "Feb", "Mar", "Apr", "May", "Jun"]}
  series={[
    { name: "Revenue", data: [820, 932, 901, 934, 1290, 1330] },
  ]}
/>
```

**Two smooth lines comparing years:**

```tsx
<LineChart
  title="Sales Comparison"
  xLabels={["Jan", "Feb", "Mar", "Apr", "May", "Jun"]}
  series={[
    { name: "2024", data: [820, 932, 901, 934, 1290, 1330], smooth: true },
    { name: "2025", data: [620, 732, 1101, 1134, 1190, 1530], smooth: true },
  ]}
/>
```

**Area chart with custom color:**

```tsx
<LineChart
  title="Website Traffic"
  xLabels={["Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun"]}
  series={[
    {
      name: "Visitors",
      data: [150, 230, 224, 218, 135, 147, 260],
      smooth: true,
      showArea: true,
      areaOpacity: 0.2,
      color: "#5470c6",
    },
  ]}
/>
```

**Small compact chart without legend:**

```tsx
<LineChart
  xLabels={["Mon", "Tue", "Wed", "Thu", "Fri"]}
  series={[{ name: "Errors", data: [5, 12, 3, 8, 2], color: "#ee6666" }]}
  variant="compact"
  showLegend={false}
  height={200}
  width={350}
/>
```

### BarChart

Vertical bars, horizontal bars, grouped comparisons, or stacked bars.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | — | Chart title at the top center |
| `xLabels` | `string[]` | **required** | Category labels |
| `series` | `BarSeriesConfig[]` | **required** | One or more data series (see below) |
| `horizontal` | `boolean` | `false` | Display bars horizontally |
| `showTooltip` | `boolean` | `true` | Show info popup on hover |
| `showLegend` | `boolean` | `true` | Show the legend |
| `variant` | `"default"` / `"compact"` / `"dashboard"` | `"default"` | Layout preset |
| `colors` | `string[]` | built-in palette | Override the color palette |
| `height` | `number` or `string` | `400` | Chart height |
| `width` | `number` or `string` | `"100%"` | Chart width |
| `loading` | `boolean` | `false` | Show a loading spinner |
| `className` | `string` | — | CSS class for the container |

#### Series configuration (BarSeriesConfig)

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `string` | **required** | Label for this series |
| `data` | `number[]` | **required** | Values — must match the length of `xLabels` |
| `color` | `string` | auto | Bar color |
| `stack` | `string` | — | Stack group name. Bars with the **same** `stack` value are stacked on top of each other |

#### Examples

**Simple bar chart:**

```tsx
<BarChart
  title="Quarterly Sales"
  xLabels={["Q1", "Q2", "Q3", "Q4"]}
  series={[
    { name: "Sales", data: [320, 302, 341, 374] },
  ]}
/>
```

**Grouped bars (side by side):**

```tsx
<BarChart
  title="Department Budget"
  xLabels={["Engineering", "Design", "Marketing", "Sales"]}
  series={[
    { name: "2024", data: [150, 80, 120, 90] },
    { name: "2025", data: [180, 95, 110, 105] },
  ]}
/>
```

**Stacked bars with custom colors:**

```tsx
<BarChart
  title="Issue Tracker"
  xLabels={["Jan", "Feb", "Mar", "Apr", "May"]}
  series={[
    { name: "Open",        data: [12, 15, 8, 10, 6],  stack: "issues", color: "#ee6666" },
    { name: "In Progress", data: [5, 8, 12, 7, 4],    stack: "issues", color: "#fac858" },
    { name: "Closed",      data: [20, 18, 25, 30, 35], stack: "issues", color: "#91cc75" },
  ]}
/>
```

**Horizontal bars:**

```tsx
<BarChart
  title="Team Performance"
  xLabels={["Alice", "Bob", "Carol", "Dave"]}
  series={[{ name: "Score", data: [85, 72, 93, 68] }]}
  horizontal={true}
/>
```

### PieChart

Pie charts and donut (ring) charts.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | — | Chart title at the top center |
| `data` | `PieSlice[]` | **required** | Slices (see below) |
| `donut` | `boolean` | `false` | Show as donut (ring) instead of filled pie |
| `showTooltip` | `boolean` | `true` | Show info popup on hover |
| `showLegend` | `boolean` | `true` | Show legend at the bottom |
| `colors` | `string[]` | built-in palette | Override the color palette |
| `height` | `number` or `string` | `400` | Chart height |
| `width` | `number` or `string` | `"100%"` | Chart width |
| `loading` | `boolean` | `false` | Show a loading spinner |
| `className` | `string` | — | CSS class for the container |

#### Slice configuration (PieSlice)

| Field | Type | Description |
|-------|------|-------------|
| `name` | `string` | Label for this slice |
| `value` | `number` | Numeric value (percentages are calculated automatically) |

#### Examples

**Basic pie chart:**

```tsx
<PieChart
  title="Browser Market Share"
  data={[
    { name: "Chrome",  value: 65 },
    { name: "Safari",  value: 18 },
    { name: "Firefox", value: 8 },
    { name: "Edge",    value: 5 },
    { name: "Other",   value: 4 },
  ]}
/>
```

**Donut chart:**

```tsx
<PieChart
  title="Revenue by Region"
  data={[
    { name: "Europe",        value: 42 },
    { name: "North America", value: 31 },
    { name: "Asia",          value: 18 },
    { name: "Other",         value: 9 },
  ]}
  donut={true}
/>
```

**Donut with custom colors:**

```tsx
<PieChart
  title="Project Status"
  data={[
    { name: "Completed",   value: 45 },
    { name: "In Progress", value: 30 },
    { name: "Planned",     value: 25 },
  ]}
  donut={true}
  colors={["#91cc75", "#fac858", "#73c0de"]}
/>
```

## Colors and themes

### Default color palette

All charts use this palette when no custom colors are specified:

| # | Color | Hex |
|---|-------|-----|
| 1 | Blue | `#5470c6` |
| 2 | Green | `#91cc75` |
| 3 | Yellow | `#fac858` |
| 4 | Red | `#ee6666` |
| 5 | Cyan | `#73c0de` |
| 6 | Teal | `#3ba272` |
| 7 | Orange | `#fc8452` |
| 8 | Purple | `#9a60b4` |
| 9 | Pink | `#ea7ccc` |

### Overriding colors

**Per series** — set `color` on an individual series:

```tsx
series={[{ name: "Revenue", data: [...], color: "#ff0000" }]}
```

**Whole palette** — pass the `colors` prop:

```tsx
<LineChart colors={["#264653", "#2a9d8f", "#e9c46a", "#f4a261", "#e76f51"]} ... />
```

### Changing the global default palette

Edit `src/utils/theme.ts` — the `DEFAULT_COLORS` array.

## Layout variants

Every chart supports a `variant` prop with three presets:

| Variant | Font size | Spacing | Best for |
|---------|-----------|---------|----------|
| `"default"` | 12px | Generous | Full-page charts, reports |
| `"compact"` | 10px | Tight | Small widgets, cards |
| `"dashboard"` | 11px | Balanced | Dashboard panels, grids |

```tsx
<LineChart variant="compact" ... />
```

To adjust the presets, edit `src/utils/theme.ts` — the `VARIANT_CONFIG` object.

## How to add a new chart type from ECharts

### The easy way — let the AI do it

1. Go to https://echarts.apache.org/examples/en/
2. Click on any chart you like
3. Click the **TS** tab (not JS)
4. Copy the entire code
5. Paste it into the Cursor chat with a message like this:

> **Create a new component from this ECharts example:**
>
> ```
> option = {
>   xAxis: {
>     type: 'category',
>     data: ['Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat', 'Sun']
>   },
>   yAxis: {
>     type: 'value'
>   },
>   series: [
>     {
>       data: [820, 932, 901, 934, 1290, 1330, 1320],
>       type: 'line',
>       smooth: true
>     }
>   ]
> };
> ```

The AI will automatically:
- Create the component file with proper props
- Export it from the index
- Create a Storybook story
- Make all hardcoded values configurable

That's it. Run `npm run dev` to see it in Storybook.

### Important: always use the TS tab

On the ECharts examples page, there are two tabs above the code: **JS** and **TS**.
Always click **TS** before copying. This matches our project's TypeScript setup and
gives the AI better type information to work with.

### Manual approach (if you prefer)

If you want to do it yourself instead of using the AI:

**1. Create the component** at `src/components/charts/YourChart.tsx`:

```tsx
import { useMemo } from "react";
import { BaseChart } from "./BaseChart";
import { DEFAULT_COLORS } from "../../utils/theme";
import type { ChartSize } from "../../types/chart";
import type { EChartsOption } from "echarts";

export type YourChartProps = ChartSize & {
  title?: string;
  // add your props here
  className?: string;
  loading?: boolean;
};

export function YourChart({
  title,
  className,
  loading,
  width,
  height,
}: YourChartProps) {
  const option = useMemo<EChartsOption>(() => {
    return {
      // paste the option object here, replace hardcoded values with props
      title: title ? { text: title, left: "center" } : undefined,
    };
  }, [title]);

  return (
    <BaseChart
      option={option}
      className={className}
      loading={loading}
      width={width}
      height={height}
    />
  );
}
```

**2. Export it** — add to `src/components/charts/index.ts`:

```tsx
export { YourChart } from "./YourChart";
export type { YourChartProps } from "./YourChart";
```

**3. Create a story** at `stories/YourChart.stories.tsx`:

```tsx
import type { Meta, StoryObj } from "@storybook/react-vite";
import { YourChart } from "../src/components/charts/YourChart";

const meta: Meta<typeof YourChart> = {
  title: "Charts/YourChart",
  component: YourChart,
  args: { height: 400, width: 600 },
};

export default meta;
type Story = StoryObj<typeof YourChart>;

export const Basic: Story = {
  args: {
    title: "My New Chart",
  },
};
```

**4. Verify** — run `npm run dev` and check Storybook.

## Using this library in another app

After building (`npm run build`), other React projects can import these components.

### Option 1: Local path (simplest)

In your other project, run:

```bash
npm install ../path/to/components
```

Then in code:

```tsx
import { LineChart, BarChart, PieChart } from "@eproject/components";

function Dashboard() {
  return (
    <div>
      <LineChart
        title="Monthly Revenue"
        xLabels={["Jan", "Feb", "Mar", "Apr"]}
        series={[{ name: "Revenue", data: [820, 932, 901, 1290] }]}
      />
    </div>
  );
}
```

### Option 2: Git repository

```bash
npm install git+https://github.com/your-org/components.git
```

### Option 3: Publish to npm

```bash
npm publish
```

Then anywhere:

```bash
npm install @eproject/components
```

The consuming app must have `react` and `react-dom` installed (v18 or v19).

## Troubleshooting

### `npm install` fails

- Make sure Node.js v18+ is installed: `node --version`
- Delete `node_modules` and `package-lock.json`, then run `npm install` again

### Storybook doesn't start

- Make sure port 6006 is free
- Try `npx storybook dev -p 6007` for a different port
- Delete `node_modules/.cache` and try again

### Charts show as blank

- The chart needs a defined `height` (default is `400`)
- Check the browser console (F12) for errors

### Build fails

- Run `npm run lint` first to find TypeScript errors
- Fix them, then try `npm run build` again

### "Cannot find module @eproject/components"

- Run `npm run build` in this project first
- Check that the path in `npm install ../components` is correct
- The consuming app needs `react` and `react-dom` installed
